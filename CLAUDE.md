## What This Is

A Terraform module that manages SAP BTP role collection assignments at the subaccount level. It accepts lists of role-collection-to-group and role-collection-to-user mappings and flattens them into individual `btp_subaccount_role_collection_assignment` resources using `for_each` with composite keys (`{role_collection}__{assignee}__{origin}`).

**Provider:** SAP/btp ~> 1.22.0 | **Terraform:** >= 1.14

## Architecture

```
.                                        # Root module (published)
├── main.tf                              # locals flatten inputs + two for_each resources (by_group, by_user)
├── variables.tf                         # subaccount_id, default_origin_id, role_collection_assignments_by_group/user
├── outputs.tf                           # assignment counts and keys per type
├── versions.tf                          # provider/terraform version constraints
├── README.md                            # Auto-generated via terraform-docs (do not edit between TF_DOCS markers)
├── examples/
│   ├── role-collection-assignment-by-group/   # Example: group-based assignments
│   │   ├── main.tf, variables.tf, outputs.tf, versions.tf, providers.tf
│   │   └── README.md
│   └── role-collection-assignment-by-user/    # Example: user-based assignments
│       ├── main.tf, variables.tf, outputs.tf, versions.tf, providers.tf
│       └── README.md
└── tests/
    └── validation_unit_test.tftest.hcl  # Native tests with mock provider (11 scenarios)
```

The core pattern: input lists (`role_collection_assignments_by_group`, `role_collection_assignments_by_user`) are double-flattened in `locals` to produce one map entry per assignment. The map key encodes role collection name, assignee name, and origin (with special characters replaced by underscores) to give stable `for_each` keys that prevent drift on list reordering.

## Conventions

- Follow the HashiCorp Terraform style guide: 2-space indent, aligned `=` signs, `snake_case` names, `for_each` over `count`, `#` comments (not `//`).
- Resource names are descriptive (`by_group`, `by_user`) since the module contains two resources.
- The `coalesce()` function provides per-assignment origin override with fallback to `default_origin_id`.
- README.md content between `<!-- BEGIN_TF_DOCS -->` and `<!-- END_TF_DOCS -->` is auto-generated by `terraform-docs` — do not edit manually. Run `terraform-docs markdown table --output-file README.md --output-mode inject .` to regenerate.
- Example README files follow the same `terraform-docs` injection pattern.
- Version constraints use `>= ` with a space after the operator and `~> ` for pessimistic constraints.
- Test email addresses use RFC 2606 reserved domains (`@example.com`).
